# 비주얼 팩토리 (Visual Factory)

> 상품 이미지 하나를 입력받아 채널별 비주얼 에셋 풀세트를 자동 설계하는 생산 라인.
> 배경 방향, 모델 스타일링, 코디 조합, PDP 카피까지 채널 스펙에 맞게 일괄 생성한다.

## 기존 스킬과의 관계

| 연계 스킬 | 관계 |
|----------|------|
| `design-generator` | 디자인 시안을 에셋 생산 입력으로 수신 |
| `visual-generation` | 각 컷 유형별 이미지 생성 프롬프트 실행 |
| `copywriting` | PDP·SNS 카피 생성 연계 (옵션) |
| `visual-content` | 숏폼·캠페인 비주얼 기획과 연결 |

## 언제 사용

- 신상품 발매 전 채널 등록용 에셋 일괄 기획 시
- 씨빅 패딩 빅IMC — 무신사·자사몰·SNS 에셋 동시 생산 시
- 콜라보 캡슐 런칭 에셋 플래닝 시
- 시즌 별 룩북 에셋 구성 시

---

## 실행 절차

### Step 1: 에셋 요청서 작성

```
1. 대상 아이템:
   - 아이템명 + 품번
   - 라인 (COVERNAT/AUTHENTIC/COLLEGE)
   - BTA 포지셔닝 (Basic/Trend/Accent)

2. 채널 선택 (복수 선택 가능):
   - 무신사 PDP
   - 자사몰 PDP
   - 인스타그램 피드
   - 인스타그램 스토리
   - 카카오 채널
   - 룩북
   - 오프라인 VM (포스터·행거POP)

3. 비주얼 스타일:
   - 스튜디오 클린 / 스트릿 로케이션 / 에디토리얼
   - 모델 ON / 플랫레이 / 제품 단독

4. 카피 포함 여부 (copywriting 스킬 연동)
```

---

### Step 2: 채널 스펙 로드

`channels.json`에서 채널별 이미지 규격을 확인한다.

**주요 채널 스펙:**
| 채널 | 해상도 | 배경 | 요구사항 |
|------|--------|------|---------|
| 무신사 PDP | 860×1148px | 화이트 | 모델 정면, 디테일 컷 필수 |
| 자사몰 히어로 배너 | 1920×800px | 브랜드 컬러 또는 화이트 | 카피 텍스트 여백 확보 |
| 인스타그램 피드 | 1080×1080px | 자유 | 브랜드 무드 일관성 |
| 인스타그램 스토리 | 1080×1920px | 자유 | 상단/하단 안전영역 준수 |
| 룩북 | 2480×3508px | 에디토리얼 자유 | 최소 3피스 완성 스타일링 |

---

### Step 3: 비주얼 방향 설계

`visual-identity.json`, `tone-manner.json`, `personas.json` 기반으로 촬영/생성 방향을 설정한다.

**커버낫 비주얼 방향:**
- 무드: Timeless, Vintage-inspired, Clean Street, Authentic
- 조명: 자연광 기반, 밝고 맑은 분위기 또는 그레인 질감 허용
- 모델: 친근하고 자연스러운 표정·포즈, K-컬처 감도 캐스팅
- 배경: 도시·일상 로케이션, 과도한 세트 연출 지양

**BTA별 비주얼 원칙:**
| BTA | 비주얼 방향 | 핵심 메시지 |
|-----|-----------|-----------|
| Basic | 악센트 컬러 중심 노출 — 안정매출 라인의 컬러 감도 제고 | "이 컬러, 지금이 기회" |
| Trend | 컬러 무관, 실루엣·소재·그래픽 중심 시각화 | "이번 시즌 필수" |
| Accent | 필수 노출 — 아이캐칭 포인트, 시즌 무드 대표 | "지금만 있는 것" |

**3B 착장 절대 금지:** 베이직 아이템 + 베이직 컬러 + 베이직 로고 착장 화보 금지
**최소 3피스 스타일링:** 상·하·아우터 또는 상·하·ACC 완성 스타일링

---

### Step 4: 에셋 매트릭스 생성

아이템 × 채널 × 컷 유형의 조합표를 작성한다.

**컷 유형 분류:**
| 유형 | 설명 |
|------|------|
| 대표 메인컷 | 정면 모델, 전신, 아이템 주력 노출 |
| 스타일링컷 | 코디 완성, 3피스 이상 |
| 디테일컷 | 소재·기법·로고 근접 |
| 단독컷 | 플랫레이 또는 행어 |
| 분위기컷 | 로케이션·라이프스타일 |
| 악센트컷 | 악센트 아이템 단독 메인 (BTA Accent 시 필수) |

**에셋 수량 가이드:**
| 채널 | 최소 | 표준 | 풀셋 |
|------|------|------|------|
| 무신사 PDP | 5컷 | 8컷 | 12컷+ |
| 자사몰 PDP | 4컷 | 8컷 | 12컷+ |
| 인스타그램 | 2컷 | 4컷 | 8컷+ |
| 룩북 | 3컷 | 6컷 | 10컷+ |

---

### Step 5: 이미지 생성 프롬프트 엔지니어링

각 에셋에 대해 `visual-generation` 스킬에 전달할 프롬프트를 작성한다.

**모델 착장 컷 프롬프트 구조:**
```
fashion photography, [아이템명] outfit, Korean model,
natural light, [로케이션: urban street / clean studio / editorial],
[시즌 무드 키워드], casual and authentic styling,
[컬러 조합], Stussy/Lemaire inspired mood, timeless casual,
[채널 스펙: 비율/배경], professional fashion photography
```

**플랫레이/단독 컷 프롬프트 구조:**
```
product flat lay, [아이템명],
[컬러: HEX], clean white background,
minimal styling, brand photography,
[디테일 포인트: 로고/소재/기법], professional product shot
```

---

### Step 6: 카피 통합 (옵션 — copywriting 연동)

**PDP 카피 구조** (`tone-manner.json` 기준):
```
헤드라인: [아이템 핵심 가치, 15자 이내]
서브 헤드: [착용 가치·스타일링 포인트, 30자 이내]
상품 설명: [소재·기능·디테일 3줄 이내]
스타일링 팁: [코디 제안 1줄]
```

**SNS 캡션 구조:**
```
[무드 도입 1줄] + [아이템 소개] + [스타일링 제안] + [해시태그]
```
