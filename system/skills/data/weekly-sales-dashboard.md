## 역할 & 목적

**전사 3개 브랜드(커버낫·커버낫·리)의 주간 판매 실적을 하나의 대시보드에 통합하고,
QR(Quantity Reorder) 자동 알림으로 리오더 타이밍을 놓치지 않게 합니다.**

- 매주 주간 엑셀 데이터를 입력하면 자동으로 HTML 대시보드를 생성합니다
- 브랜드 탭 전환으로 개별 현황 + 전사 종합 뷰를 제공합니다
- QR 알림 임계값을 기준으로 즉시 리오더가 필요한 SKU를 자동 플래그합니다

## Phase 0 — 데이터 수집

### 0-1. 입력 데이터 형식

사용자가 업로드하거나 경로를 지정하는 파일:

```
weekly/data/
├── sheet_sales-review_[브랜드]_[wNN].xlsx   ← 주간 판매 리뷰 (브랜드별)
├── sheet_product-master_[브랜드]_[wNN].xlsx ← 상품 마스터 (브랜드별)
└── sheet_qr-stock_[브랜드]_[wNN].xlsx      ← 재고 현황 (선택, 있으면 처리)
```

**브랜드 코드:**
| 브랜드 | 코드 | 파일 접두 |
|--------|------|---------|
| 커버낫 | `ww` | `ww_` |
| 커버낫 | `cv` | `cv_` |
| 리 | `lee` | `lee_` |

### 0-2. 파일이 없을 때

파일이 일부만 있거나 없는 브랜드는:
- 해당 브랜드 탭을 "데이터 없음" 상태로 표시
- 나머지 브랜드는 정상 처리
- 없는 데이터에 대해 추정치나 임의 값을 사용하지 않음

## Phase 1 — 데이터 파싱 & 처리

### 1-1. 판매 리뷰 시트에서 추출할 항목

각 브랜드 `sheet_sales-review` 파일에서:

```
[주간 판매 요약]
- 주차 (wNN)
- 기간 (YYYY-MM-DD ~ YYYY-MM-DD)
- 목표 매출 / 실판 매출 / 달성률
- 전주 대비 증감 (금액, %)
- 전년 동기 대비 증감 (있으면)

[채널별 판매]
- FSS(직영) / 자사몰 / 무신사 / 29CM / 에이블리 / 글로벌 / 기타
- 각 채널: 목표, 실판, 달성률, 전주 대비

[카테고리별 판매]
- 유니섹스 / 우먼스 / 용품 (브랜드별 카테고리 상이할 수 있음)
- 각 카테고리: 실판, 비중, 전주 대비

[SKU 판매 현황]
- 전체 운영 SKU 수
- TOP 10 판매 SKU (품번, 품명, 판매량, 판매율, 판매금액)
- BOTTOM 10 (판매 부진, 재고 소화 위험)
- 신규 입고 SKU (이번 주 신규)
```

### 1-2. 상품 마스터 시트에서 추출할 항목

```
[SKU 재고 현황] (QR 알림 기준)
- 품번 / 품명 / 카테고리 / 시즌
- 초기 발주량 / 누적 판매량 / 현재 재고량
- 판매율 = 누적판매 / 초기발주
- 주당 판매 속도 (최근 3주 평균)
- 예상 소진 주수 = 현재재고 / 주당판매속도
```

### 1-3. QR 알림 판단 로직

아래 조건 중 하나라도 해당하면 QR 플래그 설정:

| 조건 | 알림 레벨 | 의미 |
|------|---------|------|
| 판매율 ≥ 80% | 🔴 긴급 | 즉시 리오더 필요 |
| 판매율 ≥ 65% AND 예상소진 ≤ 4주 | 🟠 주의 | 이번 주 내 발주 결정 |
| 판매율 ≥ 50% AND 주당판매속도 ↑ | 🟡 검토 | 다음 주 판단 |
| 예상소진 ≤ 2주 (판매율 무관) | 🔴 긴급 | 재고 임박 |

**QR 제외 조건:**
- 시즌오프 예정 (종료 4주 이내) → QR 검토 불필요, 대신 할인 검토 권고
- 품질 이슈 마크 있는 SKU → 홀드 표시
- SKIP 등급 SKU → 자동 제외

## Phase 2 — 대시보드 HTML 생성

### 2-1. 대시보드 구조

```
┌──────────────────────────────────────────────────────┐
│  HEADER: 비케이브 전사 주간 판매 대시보드  [wNN / 날짜]  │
│  [탭: 전사종합 | 커버낫 | 커버낫 | 리]               │
├──────────────────────────────────────────────────────┤
│  🚨 QR 알림 배너: 긴급 N건 / 주의 N건 / 검토 N건       │
├─────────────┬──────────────────────────────────────  │
│  매출 KPI   │  채널별 판매 현황 (바 차트)               │
│  (스탯 카드)│                                         │
├─────────────┴──────────────────────────────────────  │
│  TOP 10 SKU                │  QR 알림 상세 목록        │
│  (테이블 + 판매율 바)       │  (긴급→주의→검토 순)      │
├─────────────────────────────────────────────────────-│
│  카테고리 믹스              │  주간 인사이트 & 액션       │
└──────────────────────────────────────────────────────┘
```

### 2-2. 탭 구조

- **전사 종합 탭**: 3개 브랜드 합산 + 브랜드별 매출 비중 차트
- **브랜드별 탭**: 해당 브랜드 전체 상세 데이터
- 탭 전환 시 URL hash 변경 (`#ww`, `#cv`, `#lee`, `#total`)

### 2-3. HTML 생성 규칙

**디자인 시스템:**
- 배경: `#FFFFFF` (라이트 모드 — 프로젝터·공유 대응)
- 폰트: `Pretendard Variable` (한글) + `Plus Jakarta Sans` (영문/숫자)
- 메인 컬러: `#1B2559` (네이비) + `#2563EB` (블루)
- 알림 컬러: 🔴`#EF4444` 🟠`#F59E0B` 🟡`#EAB308`
- 브랜드 컬러:
  - 커버낫: `#2563EB` (블루)
  - 커버낫: `#1B2559` (네이비)
  - 리: `#10B981` (그린)

**필수 컴포넌트:**

```
[1] 스탯 카드 (KPI)
- 실판 매출 / 목표 / 달성률 / 전주대비
- 숫자는 억원 단위 (소수점 2자리)
- 달성률에 색상 코딩: ≥100% 초록, ≥90% 파랑, <80% 빨강

[2] 채널 바 차트
- 가로 바 차트 (목표 vs 실판)
- 채널별 달성률 % 표시
- 전주 대비 ↑↓ 아이콘

[3] QR 알림 카드
- 알림 레벨별 색상 (긴급/주의/검토)
- 품번 / 품명 / 판매율 / 예상소진주수 / 리오더 권고수량
- 클릭 시 해당 SKU 상세 팝업 (선택사항)

[4] TOP 10 SKU 테이블
- 판매율 인라인 프로그레스 바
- 판매율 ≥ 65% = 강조 표시 (QR 대상)
- 정렬: 판매율 내림차순 기본

[5] 주간 인사이트 섹션
- 잘 된 것 (Top 3)
- 주의가 필요한 것 (Bottom 3)
- 다음 주 액션 아이템 (3개)
```

**기술 요구사항:**
- 단일 HTML 파일 (외부 의존성 없음 — CDN 폰트 제외)
- 인라인 JSON 데이터 (JavaScript 변수로 임베드)
- 반응형 (데스크톱/태블릿 기준, 모바일 최소 대응)
- 프린트 CSS 포함 (`@media print`)
- 탭 전환은 순수 JS (프레임워크 없음)
- 차트는 순수 CSS + SVG (Chart.js 불필요)

## Phase 3 — QR 알림 보고서

대시보드 HTML 외에 QR 알림 요약을 마크다운으로 별도 생성:

```markdown
# QR 알림 보고서 — [wNN] / [날짜]

## 🔴 긴급 (즉시 발주 필요)
| 브랜드 | 품번 | 품명 | 판매율 | 현재재고 | 소진예상 | 권고발주량 |
|--------|------|------|--------|---------|---------|---------|

## 🟠 주의 (이번 주 내 결정)
| ... |

## 🟡 검토 (다음 주 판단)
| ... |

## 📋 액션 체크리스트
- [ ] 긴급 SKU [N]건 발주 승인
- [ ] 주의 SKU [N]건 수급팀 확인
- [ ] 검토 SKU [N]건 다음 주 재평가
```

## 자주 묻는 질문

**Q. 엑셀 시트 구조가 브랜드마다 달라요**
→ 공통 필드(매출/채널/SKU/재고)만 추출하고, 브랜드별 커스텀 필드는 해당 탭에만 표시. 구조 파악이 안 되면 먼저 사용자에게 확인.

**Q. 재고 데이터 파일이 없어요**
→ QR 알림은 판매율 기준으로만 작동 (예상소진주수 없이). 재고 파일 있으면 정밀도 향상.

**Q. 목표 매출 데이터가 없어요**
→ 달성률 산출 불가 → 해당 셀 "목표 미입력" 표시. 전주 대비만 계산.

**Q. 3개 브랜드가 아닌 1~2개만 있어요**
→ 있는 브랜드만 탭 생성. "전사 종합"은 입력된 브랜드만 합산.
